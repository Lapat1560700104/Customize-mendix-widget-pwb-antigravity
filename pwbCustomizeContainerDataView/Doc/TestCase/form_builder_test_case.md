# 🧪 เอกสารแผนการทดสอบ: ระบบจัดรูปแบบคำถามฟอร์มไดนามิก (Form Builder Test Cases)

คู่มือนี้กำหนดแผนการทดสอบระบบ (Test Plan & Test Cases) เพื่อตรวจรับและประกันคุณภาพความถูกต้องของการนำ Widget **PwbCustomizeContainerDataView** ไปใช้ในระบบสลับลำดับคำถามฟอร์มไดนามิก (Form Builder)

---

## 📋 1. รายละเอียดเกณฑ์การทดสอบ (Test Matrix)

| Test Case ID | หัวข้อการทดสอบ (Test Scenario) | วิธีทดสอบ (Testing Method) | ผลลัพธ์ที่คาดหวัง (Expected Result) |
| :--- | :--- | :--- | :--- |
| **TC-01** | การแสดงผลฟอร์มคละรูปแบบตามประเภทข้อมูล | โหลดหน้าจอที่มีคำถามประเภทต่างกัน (Text, Choice, Date) ที่ถูกผูก Visibility Rules ไว้ | หน้าจอเรนเดอร์คำถามแถวที่ 1, 2, 3 ออกมาตรงตามดีไซน์และเงื่อนไขประเภทอินพุตของแต่ละแถวอย่างสมบูรณ์ |
| **TC-02** | การลากสลับตำแหน่งคำถาม (Drag & Drop) | คลิกเมาส์ที่ปุ่มลากจับ (Drag Handle) ของแถวแรก แล้วลากลงไปวางต่อท้ายแถวที่สอง | แถวสลับตำแหน่งตามอย่างลื่นไหล เส้นบอกทิศทาง (Drop Indicator) แสดงแสงเรืองแสงสี Accent และการ์ดยกลอยขึ้นอย่างมีมิติ |

| **TC-05** | การคืนค่าลำดับเดิมในหน้าจอเมื่อรีเฟรช | ทำการปิดหน้าจอแล้วเปิดหน้าฟอร์มตัวเดิมขึ้นมาอีกครั้ง | หน้าจอเรียงตำแหน่งข้อคำถามแบบสอดคล้องตรงกับสายอักขระที่บันทึกไว้ในข้อความแม่นยำ ไม่กลับไปเรียงตามไอดีเริ่มต้น |
| **TC-06** | การควบคุมพฤติกรรมซ่อนปุ่มจับลาก (Config) | สลับการตั้งค่าพารามิเตอร์ `Drag Handle Position` ใน Studio Pro เป็น `hide` แล้วโหลดหน้าจอ | ปุ่มจับลาก SVG 6 จุดด้านซ้ายของการ์ดคำถามจะหายไป แต่ผู้ใช้งานยังสามารถจิ้มเมาส์ไปบนการ์ดโดยตรงเพื่อลากย้ายได้ตามปกติ |

## 🛠️ 2. เงื่อนไขและขั้นตอนเตรียมทดสอบ (Pre-requisites Setup)

1. **สิทธิ์การใช้งานแอปพลิเคชัน**: บัญชีที่ใช้ทดสอบต้องผ่านการอนุญาตสิทธิ์การเข้าถึงข้อมูลแบบเขียนอ่าน (Read/Write) ในระดับ Domain Model สำหรับทั้ง `FormTemplate` และ `FormQuestion`
2. **ข้อมูลตัวอย่าง (Mock Data)**: เตรียมคำถามคละประเภทประเภทละ 1 ข้อขึ้นไป เช่น:
   - ข้อ 1 (ประเภท Text): "กรุณาระบุชื่อ-นามสกุลของคุณ"
   - ข้อ 2 (ประเภท MultipleChoice): "เพศของคุณคือ?"
   - ข้อ 3 (ประเภท Date): "วันเกิดของคุณคือวันที่เท่าไร?"

## 📝 3. ขั้นตอนการดำเนินการทดสอบระดับละเอียด (Detailed Test Steps)

### ขั้นตอนที่ A: ทดสอบลากวางจัดเรียงคำถาม
1. ล็อคอินเข้าสู่ระบบเว็บบอร์ดออกแบบฟอร์ม
2. สังเกตลำดับข้อคำถามเริ่มต้นในรายการ (เช่น ข้อ 1 -> ข้อ 2 -> ข้อ 3)
3. เล็งเมาส์ไปคลิกค้างที่ปุ่ม **Drag Handle (สัญลักษณ์ 6 จุด)** ของคำถามข้อที่ 1
4. ดึงคำถามเลื่อนลอยข้ามลงมาวางแทรกไว้ระหว่างข้อ 2 และข้อ 3
5. สังเกตแสง Glow ของเส้นบอกทิศทาง และปล่อยปุ่มคลิกเมาส์
6. **ตรวจสอบผล**: ลำดับหน้าจอจะต้องสลับสับเปลี่ยนแบบเรียลไทม์เป็น (ข้อ 2 -> ข้อ 1 -> ข้อ 3)

### ขั้นตอนที่ B: ทดสอบตรวจสอบการบันทึกค่าลงฐานข้อมูล
1. หลังจากการลากจัดเรียงในขั้นตอนที่ A เสร็จสิ้น
2. คลิกปุ่มตรวจสอบค่าหรือตรวจสอบผ่านหน้า Admin Console เพื่อดูค่าในฟิลด์ `SortedQuestionIds`
3. **ตรวจสอบผล**: ตัวแปร String จะต้องเก็บเรียงตัวเลข GUIDs คั่นจุลภาคตามลำดับจริงล่าสุดที่มองเห็นบนหน้าจอ

### ขั้นตอนที่ C: ทดสอบคืนค่าตำแหน่งลำดับ (Page Reload / Hydration)
1. ทำการกดปุ่ม F5 หรือปิดแท็บบราว์เซอร์แล้วกลับเข้าสู่หน้าจอดังกล่าวใหม่อีกครั้ง
2. **ตรวจสอบผล**: ระบบจะต้องดึงค่าลำดับจากฐานข้อมูล และสับตำแหน่งการเรนเดอร์คำถามให้ออกมาเป็น (ข้อ 2 -> ข้อ 1 -> ข้อ 3) ตรงกับการแก้ไขของเราล่าสุดอย่างแม่นยำ

### ขั้นตอนที่ D: ทดสอบการซ่อนปุ่มลากจับ (Drag Handle Position = 'hide')
1. เข้าโปรแกรม **Mendix Studio Pro** แล้วเปิดหน้าเพจที่ออกแบบ
2. ดับเบิ้ลคลิกเพื่อเปิด Properties ของตัว Widget
3. ไปที่หัวข้อ **Aesthetics** -> ปรับ `Drag Handle Position` ให้เป็น **`Hide Drag Handle`**
4. บันทึกและสั่งรันโครงการใหม่ (Run Locally)
5. **ตรวจสอบผล**: ปุ่มสัญลักษณ์หกจุดทางด้านซ้ายของการ์ดแถวรายการจะซ่อนตัวไปโดยสมบูรณ์ และพื้นที่แถวรายการขยายออกสวยงามเรียบร้อย โดยผู้ใช้ยังจับการ์ดแถวเพื่อย้ายตำแหน่งได้ราบรื่น
