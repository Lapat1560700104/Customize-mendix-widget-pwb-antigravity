# 📘 คู่มือเชิงลึกฉบับสมบูรณ์ (Comprehensive Manual): pwbCustomizeContainerDataView

## 📌 บทนำและเป้าหมายของวิจเจต (Overview & Purpose)

**`pwbCustomizeContainerDataView`** เป็น Mendix Pluggable Widget เกรดพรีเมียมระดับองค์กร (Enterprise-grade) ที่ได้รับการออกแบบและพัฒนาภายใต้สถาปัตยกรรมประสิทธิภาพสูง ทำหน้าที่เป็น **ตู้คอนเทนเนอร์อัจฉริยะที่สามารถรองรับการหย่อนวิจเจตเมนดิกซ์ตัวอื่นๆ ซ้อนด้านในได้อย่างอิสระ (Nested Pluggable Container)** พร้อมมีระบบสลับลำดับรายการด้วยการจับลากแบบเรียลไทม์ (**Fluid Pointer-capture Drag & Drop Reordering Engine**)

วิจเจตนี้พัฒนาขึ้นมาเพื่อแก้ไขข้อจำกัดหลายประการของบราวเซอร์และเครื่องมือมาตรฐานเดิม:
1.  **ขจัดข้อจำกัดของ HTML5 Drag & Drop ดั้งเดิม:** บนหน้าจอทัชสกรีนของสมาร์ตโฟน (โดยเฉพาะ iOS Safari) ระบบลากวางดั้งเดิมมักตอบสนองช้าและมีบั๊กติดขัด วิจเจตนี้จึงเลือกใช้ **Unified Web Pointer Events Engine** ในการจับพิกัดการลากอย่างเที่ยงตรงบนทุกแพลตฟอร์ม
2.  **ประสิทธิภาพไร้รอยต่อ (Zero Layout Shift & High Performance):** ระบบจัดเรียงลำดับการ์ดชั่วคราวในฝั่งเบราว์เซอร์เกิดขึ้นได้ทันทีอย่างนุ่มนวล โดยจะบันทึกผลลัพธ์กลับไปยัง Mendix Database **เพียงครั้งเดียวต่อการวางสลับสำเร็จ** ป้องกันปัญหาเซิร์ฟเวอร์ค้างจากการเขียนข้อมูลบ่อยเกินไป
3.  **ความพรีเมียมและความคุ้มค่าทางประสบการณ์ผู้ใช้ (Premium UX):** มาพร้อมระบบฟิสิกส์จังหวะจับลากยกการ์ด (Scale Lift), ระบบสั่นตอบสนอง (Android Haptics), ระบบเลื่อนหน้าจออัตโนมัติเมื่อลากการ์ดชิดขอบจอ (Ergonomic Auto-Scroll) และการ์ดดีดกลับด้วยแอนิเมชันสปริงเมื่อยกเลิกคิว (Wobble Snap-Back)

### 1.1 Unified Pointer Events Engine
ตัววิจเจตเลือกจับคู่ Listener ไปที่ `onPointerDown` ของแถวไอเทม เพื่อประมวลผลคำสั่งจากทั้ง เมาส์ (Mouse), ปากกา (Stylus) และนิ้วมือ (Touch) ภายใต้ API เดียวกัน และสั่งการ `setPointerCapture` เพื่อติดตามพิกัดการลากอย่างแม่นยำแม้ผู้ใช้จะลากนิ้วออกไปนอกขอบเขตการ์ด

### 1.2 ระบบเลื่อนจออัตโนมัติอัจฉริยะ (Dynamic Auto-Scrolling)
เมื่อผู้ใช้ลากการ์ดเข้าชิดขอบบนหรือขอบล่างของหน้าจอโทรศัพท์มือถือภายในรัศมี 75px ตัววิจเจตจะรันสูตรคำนวณสัดส่วนความเร็วในการเลื่อนหน้าอัตโนมัติ:
$$\text{ScrollSpeed} = \max\left(1, \min\left(25, \text{round}\left((\text{Threshold} - \text{DistanceToEdge}) \times 0.4\right)\right)\right)$$
ยิ่งลากการ์ดเข้าใกล้ขอบจอมากเท่าไร หน้าจอก็จะยิ่งเลื่อนขึ้นหรือลงให้เร็วขึ้นเท่านั้น (ขอบเขตจำกัดความเร็วที่ 1px ถึง 25px) ช่วยให้การจัดเรียงลิสต์ข้อมูลยาวๆ บนมือถือทำได้อย่างสะดวกสบายสูงสุด

### 1.3 ระบบการเข้าถึงและแป้นพิมพ์คีย์บอร์ด (Accessibility / WCAG Keyboard Support)
วิจเจตได้รับการพัฒนาให้รองรับการเข้าถึงอย่างสมบูรณ์แบบตามมาตรฐาน WCAG 2.1:
1.  **คีย์บอร์ดโฟกัส (Focus States):** แต่ละแถวไอเทมมีคุณสมบัติ `tabIndex={0}` และ `role="listitem"` ทำให้สามารถโฟกัสผ่านปุ่ม `Tab` ได้ พร้อมการแสดงเส้นกรอบโฟกัส (`:focus-visible`) สีเน้นตามธีมอย่างโดดเด่น
2.  **การลากวางด้วยแป้นพิมพ์ (Keyboard Drag & Drop):**
    *   **Spacebar / Enter:** เพื่อทำการ "คีบจับ" (Grab) หรือ "วางปล่อย" (Drop) การ์ดไอเทมที่กำลังโฟกัสอยู่
    *   **Arrow Up / Left:** เลื่อนย้ายตำแหน่งการ์ดไอเทมขึ้นด้านบนหรือไปทางซ้าย 1 ลำดับ
    *   **Arrow Down / Right:** เลื่อนย้ายตำแหน่งการ์ดไอเทมลงด้านล่างหรือไปทางขวา 1 ลำดับ
    *   **Escape:** ยกเลิกการจัดเรียง และดึงการ์ดไอเทมกลับคืนตำแหน่งเดิมก่อนลากทันที
3.  **เครื่องอ่านหน้าจอ (Screen Reader Live Announcements):** มีการติดตั้ง `div` ซ่อนสถานะพร้อมแอตทริบิวต์ `aria-live="assertive"` เพื่อส่งเสียงประกาศสถานะภาษาไทยและอังกฤษแบบเรียลไทม์ เช่น *"Grabbed [Item Title]..."*, *"Moved to position 3 of 5..."*, หรือ *"Dropped. Order updated."*

### 1.4 ระบบเลื่อนหน้าจอระดับกล่องเก็บข้อมูลซ้อน (Nested Container Auto-Scrolling)
ขจัดข้อจำกัดการเลื่อนหน้าจอผ่านพิกัดวินโดว์อย่างเดียว โดยเพิ่มกลไก **Container-Aware Auto-Scrolling**:
- ระบบจะค้นหาอีลิเมนต์ Scrollable parent ที่ใกล้ที่สุด (`overflow-y: auto` หรือ `scroll` และมี `scrollHeight > clientHeight`) ทันทีเมื่อเริ่มลากจับ
- ระหว่าง PointerMove เมื่อลากการ์ดเข้าใกล้ขอบเขตจำกัดด้านบน/ล่าง หรือซ้าย/ขวาของ Scrollable parent นั้น วิจเจตจะส่งคำสั่งเลื่อนสกロールแถบควบคุมในตัวแผงนั้นๆ โดยตรง แทนการเลื่อนทั้งหน้าเว็บเพจหลัก
- หากตรวจไม่พบกล่องซ้อน Scrollable parent ระบบจะดึงกลไก Scroll หน้าต่างหลัก (`window`) มาใช้เป็น Fallback เสมอ

### 1.5 การอัปเดตแบบคาดเดาและขจัดปัญหากระพริบ (Optimistic UI & Event Broker)
ในการลากวางการ์ดข้ามคอลัมน์คัมบัง (Cross-column drop) ปัญหาดั้งเดิมคือความไม่พร้อมกันของการอัปเดตแบบ Asynchronous บนเซิร์ฟเวอร์เมนดิกซ์ ซึ่งทำให้เกิดภาพกะพริบหรือการ์ดซ้ำซ้อนชั่วขณะ ระบบจึงออกแบบ **Optimistic UI Transition Engine** เพื่อแก้ไขปัญหานี้:
1.  **การดักจับสถานะลากผ่าน Global Registry:** ทันทีที่มีการทิ้งตัวลงกล่องเป้าหมาย วิจเจตจะสร้างสถานะคาดการณ์ชั่วคราวลงใน `window.__pwbActiveTransition` และสั่งการกระจายสัญญาณผ่าน `pwb-transition-start` Event Broker ทั่วหน้าต่างเบราว์เซอร์
2.  **การปรับมุมมองชั่วคราวฝั่ง UI:**
    *   วิจเจตคอลัมน์ต้นทางจะทำการกรองและซ่อนการ์ดชิ้นนั้นออกจากหน้าจอทันที (Optimistic Remove)
    *   วิจเจตคอลัมน์ปลายทางจะทำการฝังและแสดงการ์ดจำลองในพิกัดเป้าหมายทันที (Optimistic Insert)
3.  **การตรวจสอบเมื่ออัปเดตฐานข้อมูลสำเร็จ:** ทันทีที่ Mendix เซฟข้อมูลแอททริบิวต์เสร็จสิ้นและจ่ายสเตตัสใหม่เข้ามา ระบบจะเชื่อมต่อเข้าหาสเตตัสจริงโดยอัตโนมัติอย่างไร้รอยต่อ และเคลียร์สเตตัสจำลองทิ้งทันทีเมื่อครบ 1.2 วินาที

## 🎨 2. การจัดการสไตล์ ธีม และพรีเซ็ตสำเร็จรูป (Premium Theming Engine)

วิจเจตได้รับการปรับปรุงระบบสไตล์ให้รองรับชุดคำสั่งดีไซน์ยุคใหม่ (Modern CSS variables) และแบ่งหมวดหมู่ดีไซน์ออกเป็น **3 ธีมสำเร็จรูปพรีเมียม** เพื่อมอบความประทับใจตั้งแต่แรกเห็น:

### 2.1 รายละเอียดชุดธีมพรีเซ็ต (Theme Presets)
1.  **Default Modern Rounded (`default_rounded`):**
    *   ดีไซน์สไตล์การ์ดโค้งมนพรีเมียมมาตรฐาน กรอบเส้นขอบบางประณีต พร้อมเงาตกกระทบเบาบาง (Subtle drop shadow)
2.  **Premium Glassmorphic (`modern_glass`):**
    *   การ์ดสไตล์กระจกฝ้าโปร่งใส โดยใช้การผสมสีพื้นหลังร่วมกับค่าความโปร่งแสง (`color-mix` 65%)
    *   รันระบบเบลอพื้นหลังผ่านคุณสมบัติ `backdrop-filter: blur(14px)`
    *   ขอบการ์ดบางสีขาวสะท้อนแสงอ่อนๆ เหมาะสำหรับแอปพลิเคชันยุคใหม่ที่เน้นโทนสีสดใสหรือแนว Glassmorphism
3.  **Minimalist Flat (`minimalist_flat`):**
    *   ลบเส้นขอบข้างและเงานูนออกทั้งหมด เหลือไว้เพียงเส้นคั่นขอบล่างบางเบา (Border Bottom) เหมาะสำหรับแอปพลิเคชันสายดาร์กบอร์ดหรือลิสต์รายการคิวที่ต้องการความสะอาดตาสูงสุด ปราศจากองค์ประกอบที่รกตา
4.  **Neo-Brutalist High Contrast (`neo_brutalist`):**
    *   ดีไซน์แหวกแนวสไตล์ศิลปะขบถ (Anti-design) ขอบหนาสีดำปึ้ก (`3px solid #000000`) เงาตกกระทบยื่นเฉียงมุมแบบไร้ความฟุ้งเบลอ (`box-shadow: 5px 5px 0px 0px #000000`)
    *   เมื่อนำเมาส์ไปโฮเวอร์ การ์ดจะสไลด์ตัวเฉียงลอยขึ้นสวนทางกับทิศทางเงา (`transform: translate(-3px, -3px)`) เพิ่มมิติการกดอย่างมีลูกเล่น
    *   **ในโหมด Dark Mode:** เงาสีดำจะอัปเกรดเปลี่ยนร่างตามสีเน้นธีมหลัก (`accentColor`) มอบคอนทราสต์ที่เรืองแสงโดดเด่นในหน้าจอธีมมืด

### 2.2 การรองรับระบบ Dark Mode สมบูรณ์แบบ
ตัวสไตล์ชีตได้รับการวาง selectors แบบดักจับสัญญาณรอบทิศทาง:
```css
/* ตรวจพบจากการตั้งค่าของบราวเซอร์ระบบปฏิบัติการ */
@media (prefers-color-scheme: dark) { ... }

/* ตรวจพบจากการสลับธีมของตัวแอปพลิเคชัน Mendix (คลาสยอดนิยมบน body) */
:is(.theme-dark, .dark, .mx-dark) .pwb-drag-container { ... }
```
ระบบจะเปลี่ยนโทนสีย้อนกลับชุดตัวแปร Design Tokens ทันที (เช่น เปลี่ยนพื้นหลังหลักจาก `#ffffff` เป็นสีกรมท่าเข้ม `#1e293b` และปรับสีฟอนต์ให้อ่านง่ายอัตโนมัติ)

## ⚙️ 3. ตารางพารามิเตอร์และการตั้งค่า (XML Schema & Properties)

คุณสมบัติพารามิเตอร์ทั้งหมดใน Properties Panel ของ Mendix Studio Pro แบ่งกลุ่มข้อมูลไว้ดังนี้ค่ะ:

### 3.1 แหล่งข้อมูล (Data Source)
| Property Key | Type | Required | Default | Description |
|---|---|---|---|---|
| `itemsSource` | Datasource (List) | ✅ Yes | — | แหล่งข้อมูลรายการสำหรับการ์ดวนลูปจัดลำดับ |

### 3.2 คอนเทนต์กำหนดเอง (Custom Content)
| Property Key | Type | Required | Default | Description |
|---|---|---|---|---|
| `customItemContent` | Widgets | ✅ Yes | — | ลากวาง Mendix Widget ซ้อนด้านใน (ได้รับ Context ตามแถวการ์ดตัวลูกเสมอ) |

### 3.3 ระบบบันทึกพิกัดจัดลำดับ (Sorting Persistence)
| Property Key | Type | Required | Default | Description |
|---|---|---|---|---|
| `sortedAttribute` | Attribute (String) | ✅ Yes | — | String Attribute บนตัวพ่อ (Parent Entity) เพื่อเขียน/อ่านข้อความลำดับ GUIDs คั่นด้วยจุลภาค |
| `onSortAction` | Action | No | — | Action (Microflow / Nanoflow) ที่จะทำงานอัตโนมัติหลังปล่อยการ์ดเรียงเสร็จ |

### 3.4 สไตล์และความสวยงาม (Aesthetics)
| Property Key | Type | Required | Default | Description |
|---|---|---|---|---|
| `layoutDirection` | Enumeration | No | `vertical` | ทิศทางจัดเรียง: `vertical` (แนวตั้ง) หรือ `horizontal` (แนวนอนพันรอบ Grid) |
| `dragHandleDisplay` | Enumeration | No | `left` | ปุ่มลากจับ: `left` (แสดงปุ่ม 6 จุดทางซ้าย) หรือ `hide` (ซ่อนปุ่ม ใช้ลากจับตรงไหนของการ์ดก็ได้) |

| `itemPadding` | String | No | `12px 16px` | ขนาดขอบด้านในของการ์ดไอเทมแต่ละใบ (เช่น `12px 16px`, `16px`, `20px`) |
| `itemGap` | String | No | `12px` | ขนาดช่องว่างระยะห่างระหว่างการ์ดไอเทมแต่ละแถว (เช่น `12px`, `8px`, `24px`) |

### 3.5 กระดานคัมบังลากข้ามตู้ (Kanban Settings)
| Property Key | Type | Required | Default | Description |
|---|---|---|---|---|
| `enableKanban` | Boolean | No | `true` | ตั้งค่าเป็น `True` เพื่ออนุญาตให้การ์ดลากวางข้ามตู้วิจเจตคอลัมน์ข้างๆ ได้ |
| `dragGroup` | String | No | — | ชื่อกลุ่มคอลัมน์กระดานลากวางเดียวกันเพื่อให้ย้ายการ์ดข้ามหากันได้ (ต้องตั้งชื่อตรงกันทุกฝั่ง) |
| `columnValue` | String | No | — | ค่ารหัสสถานะของคอลัมน์นี้ สำหรับอัปเดตลงในการ์ดที่ย้ายเข้ามาตกในตู้นี้ (เช่น `'Done'`) |
| `itemColumnAttribute` | Attribute | No | — | แอททริบิวต์บนตัวการ์ดลูก (Task) ที่จะถูกวิจเจตเขียนข้อมูลสถานะทับให้ทันทีเมื่อย้ายตู้ |

## 🗄️ 4. สถาปัตยกรรมข้อมูลและการออกแบบ Domain Model

การตั้งค่าที่ดีไซน์ให้วิจเจตทำงานได้อย่างมีประสิทธิภาพสูงสุดโดยไม่ต้องไปนั่งเขียนลูปแก้หมายเลข `Index` ใน Microflow แนะนำตาม 2 โมเดลนี้ค่ะ:

### 4.1 รูปแบบการจัดคิวลิสต์เดียว (Simple Priority List Model)
เหมาะสำหรับรายการเรียงลำดับความสำคัญทั่วไป:
*   **โครงสร้าง Entity:**
    *   `Project` (Entity พ่อ/แม่หลัก) -> มีแอตทริบิวต์ `SortedTaskIds` (ชนิด String, ขนาด Unlimited)
    *   `Task` (Entity ลูกที่จะนำมาเรียง) -> มีความสัมพันธ์แบบ Many-to-One ไปหา `Project` (`Task_Project`)
*   **จังหวะเริ่มต้นข้อมูล:** ค่าในแอตทริบิวต์ `SortedTaskIds` ให้ปล่อยไว้เป็นค่าว่างเปล่า (Null) ตั้งแต่แรกได้ทันที วิจเจตจะอ่านลำดับเริ่มต้นจากฐานข้อมูลให้อัตโนมัติในตอนเปิดโหลดหน้าครั้งแรกสุด และจะรันเขียนค่าลำดับรหัส Comma-separated ให้เองเมื่อผู้ใช้เริ่มแตะลากสลับคิว

---

### 4.2 รูปแบบกระดานคัมบัง Trello (Multi-Column Kanban Model)
เหมาะสำหรับบอร์ดคัมบังเต็มรูปแบบที่มีการแบ่งคอลัมน์และลากสลับไปมาได้อย่างยืดหยุ่น:
*   **โครงสร้าง Entity:**
    *   `Board` (Entity บอร์ดหลักสูงสุด)
    *   `BoardColumn` (Entity คอลัมน์ย่อย เช่น ToDo, InProgress, Done)
        *   มีแอตทริบิวต์ `SortedTaskIds` (ชนิด String, ขนาด Unlimited) สำหรับคอลัมน์นั้นๆ บันทึกคิวลำดับของตัวเองแยกขาดจากตู้ตัวอื่น
        *   มีแอตทริบิวต์ `ColumnStatus` (String/Enum เช่น `'ToDo'`)
    *   `Task` (Entity ตัวการ์ดลูก)
        *   มีแอตทริบิวต์ `Status` (String/Enum เก็บสถานะการ์ดชิ้นนั้นปัจจุบัน)
        *   มีความสัมพันธ์ `Task_Board` และ `Task_Column`
*   **การตั้งค่าในหน้าจอ:**
    *   วาง Data View ของ `BoardColumn` ในแต่ละช่องคอลัมน์
    *   วางวิจเจตนี้ลงไปด้านใน Data View
    *   **ผูกฟังก์ชันอัปเดตอัตโนมัติ:** ในแท็บ **Kanban Settings** ให้เลือกผูก `Item Column Attribute` ไปที่ `Task.Status` และผูก `Column Value` ไปที่ `BoardColumn.ColumnStatus` 
    *   **ผลลัพธ์:** เมื่อคุณโยนการ์ดข้ามคอลัมน์ ตัววิจเจตจะทำการอัปเดตฟิลด์ `Task.Status` เป็นสถานะตู้ใหม่ให้ทันที พร้อมจัดเรียงข้อความรหัสไอดีและเขียนเซฟลงใน `BoardColumn.SortedTaskIds` ของตู้ปลายทางให้แบบอัตโนมัติโดยสมบูรณ์

## 🛠️ 5. คู่มือชีวิตการพัฒนาชิ้นงานสำหรับโปรแกรมเมอร์ (Developer Guide)

ชุดคำสั่งสำหรับผู้พัฒนาในการคอมไพล์และต่อยอดวิจเจตนี้:

### 5.1 การเปิดเว็บจำลองดีไซน์แบบเรียลไทม์ (Vite Playground)
วิจเจตนี้มีระบบ Playground ติดตั้งอยู่ในตัว นักพัฒนาสามารถเขียนโค้ดและทดลองหน้าตาของวิจเจตและพรีเซ็ตธีมสไตล์ได้ทันทีบนเว็บบราวน์เซอร์โดยไม่ต้องรันโปรแกรม Mendix หรือคอมไพล์ไฟล์ `.mpk`:
```bash
cd pwbCustomizeContainerDataView
npm install
npm run playground
```
*ระบบจะเปิดพอร์ต `http://localhost:3002/` อัตโนมัติ พร้อมมีแถบเครื่องมือด้านข้างให้คุณเลือกปรับ preset ธีม, ระยะ padding, และระยะ gap เพื่อเห็นการแสดงผลการลากวางได้ทันที*

### 5.2 การจัดระเบียบฟอร์แมตและไวยากรณ์ (Linting)
ก่อนอัปโหลดโค้ดหรือทำการบิวด์ชิ้นงานทุกครั้ง นักพัฒนาจำเป็นต้องรันคลาสจัดการความเรียบร้อยของโค้ดเพื่อให้อยู่ในมาตรฐานเดียวกัน (Prettier + ESLint):
```bash
npm run lint:fix
```

### 5.3 การบิวด์เพื่อจัดส่งเข้าโปรเจกต์ Mendix (Production Release)
รันสคริปต์นี้เพื่อคอมไพล์โค้ด มินิฟายไฟล์ จดบันทึกเวอร์ชันเวลา และส่งไฟล์ไปยังโปรเจกต์เป้าหมายของคุณโดยตรง:
```bash
npm run release
```
*ตัวสคริปต์จะลบไฟล์วิจเจต `.mpk` รุ่นเก่าทั้งหมดในโฟลเดอร์ Mendix `widgets/` ปลายทางทิ้งไปโดยอัตโนมัติ เพื่อป้องกันบั๊ก "Duplicate Widget ID" ที่เป็นปัญหายอดฮิตในโปรแกรม Mendix Studio Pro*

## ❓ 6. คำถามที่พบบ่อยและการแก้ไขจุดบกพร่อง (FAQ & Troubleshooting)

*   **ถาม: เปิดบิลด์ใน Mendix Studio Pro แล้วเจอข้อผิดพลาดแดงเด่นเตือนเรื่อง `saveDelay: Property must be required` แก้ไขอย่างไร?**
    *   **ตอบ:** ปัญหานี้ได้รับการแก้ไขเรียบร้อยแล้วในเวอร์ชันล่าสุด เนื่องจากกฎของ Mendix กำหนดให้ตัวแปรประเภท Integer หรือตัวเลขทั้งหมดในไฟล์คุณสมบัติ XML จำเป็นต้องตั้งค่าคุณสมบัติความบังคับกรอกข้อมูลเป็น `required="true"` เท่านั้น โดยคุณสามารถตรวจสอบความถูกต้องของบรรทัดนี้ได้ในไฟล์ [PwbCustomizeContainerDataView.xml](file:///Users/lapat.ta/Desktop/ETC%20Project/Customize-mendix-widget-pwb-antigravity/pwbCustomizeContainerDataView/src/PwbCustomizeContainerDataView.xml#L116)
*   **ถาม: ยูสเซอร์ลากการ์ดสลับที่บนมือถือไปแล้ว แต่อยู่ดีๆ บอร์ดเด้งการ์ดกลับมาตำแหน่งเดิมพร้อมการสั่นกะพริบแสงสีแดง เกิดจากอะไร?**
    *   **ตอบ:** นี่คือฟังก์ชัน **Wobble Error Rollback** หากแอตทริบิวต์ String ที่ใช้ผูกเรียงลำดับถูกตั้งค่าความปลอดภัยในฝั่ง Mendix เป็นแบบ **Read-Only** (อ่านได้แต่เขียนไม่ได้) หรือตัวแปรถูกสั่งล็อกจากขั้นตอนตรวจสอบ Validation บน Microflow ตัวรันไทม์จะทำการบล็อกไม่ให้มีการเซฟ และสั่งเด้งการ์ดดึงกลับที่เดิมพร้อมสั่นเพื่อแจ้งเตือนให้ผู้ใช้งานทราบทันทีค่ะ
*   **ถาม: ขนาดความกว้างของการ์ดเพี้ยนไปหรือหน้าจอยืดหดไม่คงที่เมื่อนำไปใส่ข้อมูลคัมบังแนวนอน ทำอย่างไรดี?**
    *   **ตอบ:** คุณสามารถใช้ Properties `itemPadding` และ `itemGap` ที่เราพัฒนาเพิ่มเติมใหม่ในการกำหนดขนาดช่องไฟให้พอดีกับความกว้างหน้าจอการแสดงผล หรือสามารถเขียนแทรกคำสั่งคลาสทับได้โดยตรงที่เป้าหมายตัวการ์ด `.pwb-draggable-row-item` ในไฟล์ CSS หลักค่ะ
