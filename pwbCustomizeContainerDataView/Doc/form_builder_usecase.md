# 📋 กรณีใช้งาน: ระบบจัดรูปแบบคำถามฟอร์มไดนามิก (Dynamic Form Builder Use Case)

---

## 📌 1. ภาพรวมของระบบ (Overview)

ในระบบงานองค์กรหรือการทำแบบสอบถามออนไลน์ มักมีความจำเป็นต้องออกแบบฟอร์มตอบคำถามที่มี **คำถามหลากหลายรูปแบบคละกันอยู่ในหน้าเดียว** (เช่น ข้อความสั้น, ข้อความยาว, เลือกตอบวิทยุ, หลายตัวเลือก, และปฏิทินวันที่) และผู้สร้างแบบฟอร์มต้องการสลับตำแหน่ง ลำดับข้อคำถาม (Question Reordering) ได้ตามต้องการ

**`PwbCustomizeContainerDataView`** ทำหน้าที่เป็นตัวสลับลำดับอัจฉริยะ โดยเปิดให้นำ Mendix Widget ที่แตกต่างกันมาประกอบเข้าด้วยกันในแต่ละแถวรายการ (Row Item) และสลับตำแหน่งได้แบบเรียลไทม์ ทั้งผ่านเมาส์/สัมผัส และผ่านคีย์บอร์ดเพื่อรองรับผู้ทุพพลภาพ (WCAG Compliance)

---

## 🏗️ 2. สถาปัตยกรรมระดับออบเจกต์ (Object Architecture & Domain Model)

ในการทำ Form Builder ระบบจะสร้างความสัมพันธ์ดังนี้:

```
┌──────────────────────────────────────┐          ┌──────────────────────────────────────┐
│             FormTemplate             │          │             FormQuestion             │
│   (Context Entity ของหน้าหลัก)         │          │      (Datasource - แถวไอเทมคำถาม)     │
├──────────────────────────────────────┤          ├──────────────────────────────────────┤
│ - TemplateName (String)              │ 1      * │ - QuestionText (String)              │
│ - SortedQuestionIds (Unlimited String)◄─────────┼─ - QuestionType (Enum: Text, Choice) │
│                                      │          │ - SectionGroup (String/Enum)         │
└──────────────────────────────────────┘          └──────────────────────────────────────┘
```

* **`FormTemplate`**: เป็นแม่พิมพ์ฟอร์มหลัก (Parent Entity) เก็บคอนฟิกชื่อฟอร์ม และบันทึกข้อความเรียงลำดับ `SortedQuestionIds` ในรูปแบบรหัส GUIDs คั่นด้วยเครื่องหมายจุลภาค (comma-separated GUID strings)
* **`FormQuestion`**: เป็นคำถามแต่ละข้อ (Child Entity) เก็บข้อมูลตัวคำถาม ประเภทของคำถาม (เช่น Text, Choice, Date) ลำดับ และฟิลด์ `SectionGroup` ในกรณีที่แบ่งส่วนคำถามหลายคอนเทนเนอร์ (เช่น ข้อมูลส่วนตัว, คำถามทั่วไป) เพื่อลากย้ายข้ามส่วนได้

## 🛠️ 3. แนวทางการจัดหน้าจอใน Mendix Studio Pro (Mendix Layout Setup)

เพื่อให้แต่ละแถวคำถามสามารถแสดงผลประเภทคำรับข้อมูลที่ต่างกันได้ ให้ทำตามขั้นตอนการวาง Layout ซ้อนด้านในช่อง **Custom Item Content** ของวิจเจต ดังนี้:

```
[ Custom Item Content Dropzone ]
  └─ [ Main Row Container ]
       ├─ [ Drag Handle Area (หากเปิดใช้แบบกำหนดเอง หรือใช้ Left Handle ในตัว) ]
       ├─ [ Header Area: แสดง QuestionText เช่น "1. กรุณากรอกอายุของคุณ" ]
       └─ [ Body Area: คอนเทนเนอร์คละสลับการเรนเดอร์คำตอบ ]
            ├─ [ Container A ] (Visible if: QuestionType = 'Text')
            │    └─ [ Mendix Standard TextBox Widget ]
            ├─ [ Container B ] (Visible if: QuestionType = 'MultipleChoice')
            │    └─ [ Mendix Standard RadioButtons or custom Dropdown Widget ]
            └─ [ Container C ] (Visible if: QuestionType = 'Date')
                 └─ [ Mendix Standard DatePicker or custom DatePicker Widget ]
```

### การตั้งค่า Conditional Visibility (การแสดงผลตามเงื่อนไข):
* **Container A (คำตอบเขียนตอบ)**: ตั้งค่าการมองเห็นให้ตรวจสอบเงื่อนไข `QuestionType = 'Text'`
* **Container B (คำตอบแบบตัวเลือก)**: ตั้งค่าให้มองเห็นเมื่อ `QuestionType = 'MultipleChoice'`
* **Container C (คำตอบแบบปฏิทินวันที่)**: ตั้งค่าให้มองเห็นเมื่อ `QuestionType = 'Date'`

## ♿ 4. ประสบการณ์ผู้ใช้ขั้นสูงและการเข้าถึง (Advanced UX & WCAG Accessibility)

โครงสร้างสถาปัตยกรรมใหม่ของวิจเจตได้รับการอัปเกรดเพื่อยกระดับความสะดวกและรองรับมาตรฐานการเข้าถึงเต็มรูปแบบ:

### 4.1 การเรียงลำดับคำถามด้วยคีย์บอร์ด (WCAG Keyboard Navigation)
ในหน้าจอผู้ดูแลระบบ (Form Builder) ผู้ใช้งานที่พึ่งพาคีย์บอร์ดเพียงอย่างเดียว (Keyboard-only users) หรือโปรแกรมอ่านหน้าจอ (Screen Reader) สามารถสลับตำแหน่งคำถามได้อย่างปลอดภัย:
* **การใช้งานคีย์บอร์ด**:
  1. ใช้ปุ่ม `Tab` เพื่อเปลี่ยนโฟกัสไปตามการ์ดคำถามต่างๆ วิจเจตจะเรืองแสงขอบโฟกัสอย่างชัดเจน
  2. กดปุ่ม `Spacebar` หรือ `Enter` เพื่อทำการ **คีบจับ (Grab)** การ์ดคำถาม
  3. กดปุ่ม **ลูกศรขึ้น/ลง (Arrow Up/Down)** เพื่อเลื่อนสลับตำแหน่งคำถามขึ้นหรือลงทีละ 1 ลำดับ
  4. กดปุ่ม `Spacebar` หรือ `Enter` อีกครั้งเพื่อทำการ **วางปล่อย (Drop)**
  5. กดปุ่ม `Escape` ได้ทุกเมื่อเพื่อยกเลิกการสลับลำดับ และย้อนคืนตำแหน่งเดิมของการ์ดทันที
* **การส่งข้อมูลเสียง (Live Announcements)**:
  ระบบจะส่งคำสั่งอ่านออกเสียงภาษาไทยและอังกฤษผ่าน `aria-live="assertive"` แจ้งเตือนผู้ใช้งาน เช่น *"คีบจับการ์ดคำถามที่ 1 แล้ว..."*, *"ย้ายไปตำแหน่งที่ 3 จาก 5..."* หรือ *"วางการ์ดเรียบร้อย ลำดับข้อมูลได้รับการอัปเดต"*

### 4.2 ระบบเลื่อนแถบข้อคำถามอัตโนมัติในช่องซ้อน (Nested Container Auto-Scrolling)
ในการทำฟอร์มขนาดใหญ่ รายการคำถามมักถูกจัดวางอยู่ในกล่องเลื่อนด้านข้าง (Scrollable Sidebar) หรือแผงควบคุมที่มีการจัดเลย์เอาต์ `overflow-y: auto`:
* **Container-Aware Scrolling**: เมื่อเริ่มลากข้อคำถามด้วยนิ้วมือหรือเมาส์ วิจเจตจะตรวจหาขอบเขตของ Scroll parent ที่ใกล้ที่สุดแบบไดนามิก
* **Auto-Scroll Proximity**: หากลากข้อคำถามไปชิดขอบบนหรือขอบล่างของกล่อง Sidebar นั้น วิจเจตจะสั่งเลื่อนสกロール (scroll) ภายในตัว Sidebar โดยตรงเพื่อเลื่อนดูข้อคำถามถัดไปทันที โดยไม่ขยับสกロールของหน้าจอหลักเบื้องหลัง ช่วยให้จัดการฟอร์มขนาดยาวบนหน้าจอมือถือหรือเว็บเดสก์ท็อปได้อย่างลื่นไหล

## 🔀 5. การจัดหมวดหมู่คำถามและการลากข้ามส่วน (Cross-Section Form Builder / Kanban Mode)

สำหรับ Form Builder ที่มีความซับซ้อน เช่น การแบ่งกลุ่มคำถามออกเป็นส่วนย่อยๆ (Sections):
1. **การกำหนดคอลัมน์ (Lanes/Columns)**: ผู้พัฒนาสามารถนำวิจเจต `PwbCustomizeContainerDataView` หลายตัวมาวางเรียงกัน โดยแต่ละตัวเป็นตัวแทนของแต่ละส่วน (Section)
2. **การตั้งค่าข้ามส่วน**:
   * ตั้งค่า `enableKanban` เป็น `true` บนทุกวิจเจต
   * กำหนดค่า `dragGroup` เช่น `"form-builder-group"` ให้ตรงกัน
   * ผูกฟิลด์ `itemColumnAttribute` เข้ากับแอตทริบิวต์ `SectionGroup` ของเอ็นทิตี `FormQuestion`
   * กำหนดค่า `columnValue` ประจำตัววิจเจตตามหมวดหมู่ (เช่น `"Section1"`, `"Section2"`)
3. **การเปลี่ยนผ่านแบบคาดเดาความเร็วสูง (Optimistic UI Transitions)**:
   เมื่อผู้ดูแลระบบลากคำถามจาก "ส่วนที่ 1" ข้ามไปวางใน "ส่วนที่ 2" ตัววิจเจตจะทำการสลับการ์ดคำถามและแสดงผลในตำแหน่งใหม่บนหน้าจอทันทีแบบไร้รอยต่อ โดยจะไม่มีปัญหาภาพกะพริบหรือการ์ดซ้ำซ้อนชั่วคราว แม้ว่าระบบ Mendix กำลังประมวลผลเซฟบันทึกฐานข้อมูลอยู่หลังบ้านก็ตาม

## ⚡ 6. การหน่วงเวลาเพื่อลดภาระเซิร์ฟเวอร์ (Performance Debouncing)

ในจังหวะการเรียงลำดับคำถามอย่างรวดเร็ว (เช่น การลากรูดสลับลำดับทีเดียวหลายช่อง):
* **ลดภาระฐานข้อมูล (Database optimization)**: การส่งคำสั่งอัปเดตลงเซิร์ฟเวอร์แบบเรียลไทม์ทุกพิกัดการลากจะทำให้เซิร์ฟเวอร์ทำงานหนักเกินจำเป็น
* **การประยุกต์ใช้ `saveDelay`**: ตั้งค่า `saveDelay` เช่น `300` หรือ `500` มิลลิวินาที ระบบจะทำการหน่วงเวลา (Debounce) หลังจากผู้ใช้ปล่อยการ์ดเป็นเวลาตามกำหนด หากผู้ใช้มีการลากสลับต่ออย่างต่อเนื่อง ระบบจะทำการเคลียร์ตัวจับเวลาเดิมและเริ่มนับใหม่ จนกระทั่งไม่มีการเคลื่อนไหวใดๆ จึงจะยิงส่งข้อมูลข้อความ GUIDs ลำดับล่าสุดและเรียกสั่งงาน `onSortAction` (Microflow/Nanoflow) เพื่ออัปเดตข้อมูลลงฐานข้อมูลจริงเพียงครั้งเดียว
