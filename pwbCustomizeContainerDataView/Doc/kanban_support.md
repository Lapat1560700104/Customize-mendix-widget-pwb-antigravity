# คู่มืออธิบายการทำงานเชิงลึก - คุณสมบัติ Kanban Support (v1.1.0)

เอกสารฉบับนี้อธิบายโครงสร้าง สถาปัตยกรรมทางเทคนิค และแนวทางการใช้งานคุณสมบัติ **Kanban Support** ของ **PWB Customize Container DataView** ซึ่งช่วยให้นักพัฒนา Mendix สามารถสร้างบอร์ดคัมบังหรือระบบลากสลับการ์ดข้ามกลุ่มได้แบบ Zero-dependency และทำงานร่วมกับ Mendix Platform ได้อย่างเต็มประสิทธิภาพ

---

## 🌟 ภาพรวมของคุณสมบัติ (Feature Overview)

ฟีเจอร์ **Kanban Support** ได้รับการพัฒนาขึ้นเพื่อเปลี่ยน Mendix Pluggable Widget แบบรายการเรียงเดี่ยว (Single Sortable List) ให้สามารถเชื่อมต่อและสร้างปฏิสัมพันธ์ร่วมกันระหว่าง Widget หลายชิ้นบนหน้าจอเดียวกันได้ ทำให้การลากการ์ดข้ามสถานะ (เช่น ลากจากคอลัมน์ **To Do** ไปยัง **In Progress**) สามารถทำงานได้โดยอัตโนมัติ

### ความสามารถหลัก:
* **Cross-Column Dragging**: ลากการ์ดสลับข้ามระหว่าง Container คนละตัวกันได้ทันทีเพียงกำหนด Group ให้ตรงกัน
* **Dynamic Property Guard (`enableKanban`)**: มีสวิตช์ควบคุมเปิด/ปิดระบบลากข้ามคอลัมน์ ทำให้นำ Widget ไปประยุกต์ใช้ได้ทั้งบอร์ด Kanban ทั่วไป และรายการแบบปิดเฉพาะตัว
* **Auto-attribute Sync**: บันทึกการเปลี่ยนแปลงสถานะหรือจัดหมวดหมู่ของการ์ดแต่ละใบลงสู่ฐานข้อมูล Mendix อัตโนมัติผ่าน Attribute ของตัวการ์ดเอง
* **Bi-directional Order Sync**: อัปเดตสตริงลำดับไอดี (`sortedAttribute`) ของทั้งคอลัมน์ต้นทาง (ลบ ID ออก) และคอลัมน์ปลายทาง (เพิ่ม ID เข้าที่ตำแหน่งที่วาง) ในจังหวะเดียว

## 📐 สถาปัตยกรรมการแชร์ข้อมูล (Data Flow & Architecture)

เนื่องจากใน Mendix Pluggable Widget แต่ละชุดที่ถูกวางลงบนเพจจะทำงานแยกขาดจากกันในระบบ React Virtual DOM (Isolated React Instances) เราจึงออกแบบ **Global Drag Registry** บนระดับเบราว์เซอร์ (`window` Object) เพื่อเป็นตัวกลางในการเชื่อมโยงข้อมูลในระหว่างลากวาง

### แผนผังกลไกการทำงาน (Sequence Diagram):

       │                                   │                                    │
 1. handleDragStart                        │                                    │
 ───►  บันทึกข้อมูลการ์ด ───────────────────►│                                    │
       และ Callback สำหรับลบการ์ด            │                                    │
       │                                   │                                    │
       │                             2. handleDragOver                          │
       │                             ◄──────────────────────────────────────────  ตรวจสอบสิทธิ์การลากผ่าน
       │                             (เช็ค dragGroup, enableKanban)             │

## ⚙️ การตั้งค่าใน Mendix Studio Pro (Properties Configuration)

นักพัฒนาสามารถเปิดใช้งานระบบลากข้ามคอลัมน์ได้โดยกำหนดค่าคุณสมบัติในแท็บ **Kanban Settings** ของตัว Widget ดังนี้:

| Property Key | Caption | Type | Required | Description |
| :--- | :--- | :--- | :---: | :--- |
| `enableKanban` | Enable Kanban Support | Boolean | ✅ | สวิตช์หลักสำหรับเปิด/ปิดการลากข้ามคอลัมน์ (ค่าตั้งต้นคือ `true`) |
| `dragGroup` | Drag Group Name | String | No | ชื่อกลุ่มของกระดานลาก (Widget ที่ต้องการลากการ์ดข้ามหากันจำเป็นต้องใช้ชื่อกรุ๊ปตรงกัน เช่น `"project-kanban"`) |
| `columnValue` | Column Value | String | No | ค่าสถานะ/หมวดหมู่ของคอลัมน์นั้นๆ (เช่น `"ToDo"`, `"InProgress"`, `"Done"`) |
| `itemColumnAttribute` | Item Column Attribute | Attribute | No | Attribute ประเภท String หรือ Enum บนตัว Entity ของแผ่นการ์ด ซึ่งจะได้รับการเปลี่ยนค่าให้เท่ากับ `columnValue` ปลายทางทันทีที่ปล่อยการ์ดลงในกลุ่มนั้น |

## 🔒 กลไกความปลอดภัยและความสอดคล้อง (Double-sided Guards)

เพื่อป้องกันไม่ให้เกิดความผิดพลาดในการสลับตำแหน่ง หรือการลากองค์ประกอบที่ไม่เกี่ยวข้องมาวางทับกัน ตัว Widget ได้วางตรรกะความปลอดภัยไว้สองด้านอย่างรัดกุม:

1. **ฝั่งคอลัมน์ต้นทาง (Source Column Guard)**:
   * หากคอลัมน์ต้นทางมีค่า `enableKanban` เป็น `false` ระบบจะล้างข้อมูล Callback สั่งย้ายไอเทม (`onRemoveItem`) และชื่อกลุ่ม (`sourceDragGroup`) ออกจาก Registry ของเบราว์เซอร์ทันทีที่เริ่มลาก
   * *ผลลัพธ์*: จะไม่มีคอลัมน์ปลายทางใด ๆ สามารถอนุญาตให้ดรอปการ์ดใบนี้ลงบอร์ดได้ (ลากเรียงตำแหน่งได้เฉพาะในกลุ่มตัวเอง)

2. **ฝั่งคอลัมน์ปลายทาง (Target Column Guard)**:
   * เมื่อมีการ์ดใด ๆ ถูกลากมาโฮเวอร์เหนือตัวคอลัมน์ ระบบจะดึงข้อมูลสถานะจาก `window.__pwbDragRegistry` ขึ้นมาวิเคราะห์
   * หากคอลัมน์ปลายทางมีค่า `enableKanban` เป็น `false` หรือมีชื่อกลุ่ม (`dragGroup`) ที่ไม่ตรงกับกลุ่มของการ์ด ตัวคอลัมน์ปลายทางจะปิดกั้นฟังก์ชัน Drop Target และแสดงสถานะไม่อนุญาตให้วาง (Blocked Hover State) ทันที

## 💎 เทคนิคการปรับปรุงประสิทธิภาพ (Performance Masteries)

การลากสลับคอลัมน์ใน Kanban มักจะเจอกับปัญหาประสิทธิภาพ (Performance degradation) เมื่อมีจำนวนการ์ดบนจอเป็นจำนวนมาก ระบบของเราแก้ไขข้อจำกัดนี้ด้วยกลยุทธ์ที่ล้ำหน้า:

* **Save Debouncing (`saveDelay`)**:
  * เมื่อลากการ์ดเปลี่ยนกลุ่ม ระบบจะไม่ยิงส่งคำสั่งเขียนค่าลง Mendix Platform และจะไม่รัน Microflow ทันทีเพื่อความลื่นไหลระดับ 60FPS
  * ระบบจะรอจนกว่าจะครบกำหนดเวลาหน่วง (Debounce Delay เช่น 300ms) หลังผู้ใช้นิ่งมือ/ปล่อยการ์ดเสร็จเรียบร้อยแล้ว จึงจะทำการบันทึกและรันโปรเซสเบื้องหลังเพียงครั้งเดียว
* **Zero-Layout-Shift via Box-Shadow**:
  * ตัวช่วยชี้จุดวางการ์ด (.pwb-drag-over) เลือกใช้งาน `box-shadow` ร่วมกับ `transform: translate` แทนการปรับขนาด border หรือ padding
  * ทำให้เบราว์เซอร์ไม่ต้องคำนวณเค้าโครงพิกเซลหน้าจอใหม่ทั้งหมด (No Layout Reflow) ส่งผลให้อัตราเฟรมเรตสม่ำเสมอ ไม่มีจังหวะภาพกระตุก
* **Absolute-Layered Empty Dropzones**:
  * ในกรณีคอลัมน์เป็นค่าว่าง (Empty State) ตัวกล่องรับการลากวาง (DragContainer) จะถูกขยายคลุมพื้นที่คอลัมน์ทั้งหมดแบบโปร่งแสงบน Layer บนสุด
  * ส่วนภาพประกอบ Empty State จะซ้อนอยู่ด้านหลัง และตั้งค่า CSS `pointer-events: none`
  * *ผลลัพธ์*: เบราว์เซอร์สามารถจับพิกัดการวางการ์ดลงคอลัมน์ว่างได้ทันที ไม่ว่าผู้ใช้จะหย่อนลงมุมใดของคอลัมน์ และไม่มีปัญหาไอคอนหรือป้ายกำกับข้อความใน Empty State ไปบล็อกการทำงานของการลากวาง
