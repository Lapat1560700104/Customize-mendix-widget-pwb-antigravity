## 🔍 รายละเอียดจุดประเมินและสถาปัตยกรรมระดับ Enterprise

### 1. การรองรับการบันทึกข้อมูลแบบ Dual-Type (DateTime & String)

* **พฤติกรรมขาเข้า (Hydration)**: เมื่อดึงค่าจากฐานข้อมูล Mendix หากพบว่าฟิลด์เป็น String ระบบจะแกะความหมายตัวเลขปี พ.ศ. หรือ ค.ศ. ผ่านกลไก Tokenizer และหักลบ Offset ปีโดยอัตโนมัติ เพื่อส่งให้ React Component วาดตารางปฏิทินได้อย่างราบรื่น
* **พฤติกรรมขาออก (Serialization)**: เมื่อนักพัฒนาผูกกับ String ฟิลด์ ตัว Widget จะนำข้อมูลวันที่แปลงเป็นข้อความ String รูปแบบเฉพาะตามที่ระบุใน **Date Format** ทันที ช่วยขจัดความจำเป็นในการเขียนโค้ด Nanoflow / Microflow เพื่อแปลงค่าแบบดั้งเดิม

### 2. ความแม่นยำของระบบแกะข้อความ (Tokenizer Parser Safety)

* มีระบบตรวจสอบการทับซ้อนของโทเค็นปฏิทินเชิงรับ ป้องกันปัญหาโทเค็นสั้น (เช่น `YY` และ `hh`) โดนแปลงซ้ำซ้อนกับตัวอักษรของปี 4 หลัก (`YYYY`) ทำให้ไม่ว่าจะใช้ Pattern รูปแบบใดก็ตาม การแปลงค่าจะมีความมั่นคง ปลอดภัย และแม่นยำสูงสุด

### 3. มาตรฐานการเข้าถึงระดับสากล (WCAG 2.1 AA)

* เพิ่มระบบนำทางด้วยแป้นพิมพ์ (Arrow Keys Navigation) มีขอบเน้นเป้าหมายที่ชัดเจน (`.pwb-day-focused`)
* หน้าจอรองรับการอ่านข้อความแบบภาษาธรรมชาติตามศักราชที่ผู้ใช้สลับใช้งาน (เช่น พุทธศักราช หรือ คริสต์ศักราช) ทำให้อุปกรณ์ช่วยเหลือผู้พิการเข้าถึงหน้าจอได้อย่างเท่าเทียม

### 4. ความทนทานในการรันบนระบบ Mendix Cloud

* โค้ดคอมโพเนนต์หลักทำงานร่วมกับแอตทริบิวต์ของ Mendix ได้อย่างปลอดภัย ผ่านการเรียกใช้งานคุณลักษณะ `.formatter` ที่รับรองโดยทีมงาน Mendix Pluggable Widgets อย่างเป็นทางการ ช่วยรับประกันว่าจะไม่มีการแจ้งเตือน Error หรือ Warning ใดๆ จากตัวคอมไพเลอร์ Rollup และ TypeScript
